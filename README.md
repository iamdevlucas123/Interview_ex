# Interview Exercises in JavaScript

Welcome to the repo where JavaScript gets grilled politely.

This project is a personal collection of coding interview exercises organized by difficulty, with a strong focus on problem solving, backend fundamentals, and the kind of questions that show up right when your confidence was starting to feel healthy.

## What's Here

Three folders, three levels of emotional damage:

- `Interview_easy` for warm-up problems and core logic practice
- `interview_medium` for problems that expect cleaner thinking and better tradeoffs
- `interview_hard` for questions that start with "design a system that..."

## Current Topics

This repo covers a mix of classic interview fundamentals:

- Arrays and strings
- Hash maps and sets
- Two pointers
- Recursion
- Intervals
- Trees from flat data
- Caching
- Event systems
- Rate limiting
- Async control and retries

## Folder Map

### `Interview_easy`

Beginner-friendly exercises such as:

- vowel counting
- largest number
- palindrome checks
- fizz buzz
- missing number
- merge sorted arrays
- a few LeetCode-style warmups

### `interview_medium`

More realistic interview problems such as:

- grouping anagrams
- flattening nested arrays
- merging intervals
- deep cloning
- LRU cache
- debounce

### `interview_hard`

Senior-style exercises with more architecture and async behavior:

- async pool
- rate limiter
- dependency scheduler
- event bus
- object diffing
- query cache with TTL
- retry with backoff

## How To Use

Pick a file. Read the prompt. Implement the solution. Realize there was an edge case. Fix the edge case. Repeat until you trust yourself again.

You can also use this repo to practice:

- explaining code in English while writing
- comparing multiple solution approaches
- improving time and space complexity
- preparing for live coding interviews

## Stack

- JavaScript
- patience
- console.log

## Good Practice Flow

1. Solve the exercise without searching first.
2. Explain your reasoning out loud in English.
3. Refactor for readability.
4. Check edge cases.
5. If the solution looks too clever, make it less suspicious.

## Why This Repo Exists

Because interview prep is easier when the exercises are organized, the goals are clear, and the README does not look like it was written during a network outage.
